# /vg-deck — 정보 밀집형 프레젠테이션 생성

> 목차/개요를 입력하면 **화면을 꽉 채우는 정보 밀집형 슬라이드**를 PPTX/PDF로 자동 생성합니다.
> `/vg-slides`(미니멀 영상)와 달리 **정보량 최대화**, **텍스트+데이터 중심**, **정적 문서 출력**이 핵심입니다.

## vg-slides와의 차이점

| 항목 | /vg-slides | /vg-deck |
|------|-----------|----------|
| 출력 형식 | MP4 영상 (Remotion) | **PPTX + PDF** |
| 스타일 | 미니멀, 센터 정렬, 키워드 중심 | **정보 밀집, 풀스크린, 문장 중심** |
| 텍스트 밀도 | 슬라이드당 50단어 이내 | **슬라이드당 100~200단어** |
| 레이아웃 | 여백 많은 프레젠테이션 | **화면 꽉 채우는 보고서형** |
| 애니메이션 | entrance/emphasis 모션 | **없음 (정적 문서)** |
| 의존성 | Remotion + React | **pptxgenjs (Node.js)** |

---

## 트리거

```
/vg-deck {제목}
/vg-deck {제목} --format pptx    # PPTX만 (기본)
/vg-deck {제목} --format pdf     # PDF만
/vg-deck {제목} --format both    # PPTX + PDF 모두
/vg-deck docs/outline.md         # 마크다운 파일 참조
```

## 입력 형식

### A. 인라인 목차

```
/vg-deck "2026 사업 보고서"

1. 표지: 2026 사업 보고서
2. 목차
3. 경영 실적 요약
   - 매출 추이 (3개년)
   - 영업이익률 변화
4. 사업부별 성과
   - 플랫폼 사업부
   - AI 솔루션 사업부
5. 핵심 KPI 대시보드
6. 내년 전략 로드맵
7. Q&A
```

### B. 마크다운 파일

```
/vg-deck docs/quarterly-report.md
```

### C. 제목만 (Codex가 목차 자동 생성)

```
/vg-deck "React 19 신기능 정리"
```

## 디자인 시스템

### 컬러 팔레트 (따뜻한 네이비 + 블루 악센트)

**모든 슬라이드가 동일한 다크 배경을 사용합니다. 라이트 배경 없음.**
**레퍼런스: 동대문정보화도서관 강의 자료 디자인**

```
BG            = #2D3142   (따뜻한 네이비 — 메인 배경)
SURFACE       = #363B50   (카드/컨테이너 표면)
SURFACE2      = #3F4560   (코드블록, 활성 상태)
ACCENT        = #4A90D9   (파란 악센트 — 제목 바, 뱃지)
ACCENT2       = #5BA0E8   (밝은 파란 — 서브헤더, 강조 텍스트)
ACCENT_DIM    = #3A6FA0   (어두운 파란 — 테이블 헤더)
CYAN          = #22D3EE   (시안 — 코드, 보조 포인트)
TEXT          = #F1F5F9   (Slate 100 — 본문)
TEXT_DIM      = #94A3B8   (Slate 400 — 부제, 설명)
TEXT_MUTED    = #64748B   (Slate 500 — 페이지 번호, 메타)
SUCCESS       = #34D399   (Emerald 400 — 콜아웃 보더, 긍정)
WARNING       = #FBBF24   (Amber 400 — 주의)
DANGER        = #F87171   (Red 400 — 부정)
BORDER        = #4A5068   (카드 테두리)
CALLOUT_BG    = #2A3040   (콜아웃/강사스크립트 배경)
CODE_BG       = #1E2233   (터미널/코드 배경)
```

### 타이포그래피

| 요소 | 폰트 | 크기(pt) | 굵기 | 색상 |
|------|------|---------|------|------|
| 슬라이드 제목 | Pretendard | 28~32 | Bold | PRIMARY 또는 WHITE |
| 섹션 헤더 | Pretendard | 20~24 | SemiBold | ACCENT |
| 본문 | Pretendard | 14~16 | Regular | TEXT_PRIMARY 또는 TEXT_DARK |
| 캡션/출처 | Pretendard | 10~12 | Regular | TEXT_SECONDARY |
| 데이터 값 | Pretendard | 36~48 | Bold | ACCENT |
| 코드 | Consolas | 12~14 | Regular | WHITE on SECONDARY |

### 레이아웃 원칙

1. **제목 위 짧은 악센트 바**: 0.7인치 파란 바가 섹션 제목 위에 위치
2. **서브헤더**: 파란색(ACCENT2) 볼드 텍스트로 소제목 표현
3. **라운드 카드**: SURFACE 배경 + BORDER 테두리 + 0.1인치 라운딩
4. **좌측 보더 콜아웃**: 강사 스크립트/실행방안 등은 좌측 컬러 보더 카드
5. **번호 뱃지**: 파란 원형 배경에 흰색 숫자
6. **도트 그리드 커버**: 좌상단 도트 패턴 장식
7. **센터형 TOC**: 시간 뱃지 + 세로 구분선 + 내용 행 구조
8. **페이지 번호**: 우하단 "N / 전체" 형식

## 워크플로우

### Step 1: 목차 분석 + 슬라이드 계획

목차의 각 항목을 분석하여 **슬라이드 유형**을 배정합니다:

```json
{
  "title": "2026 사업 보고서",
  "slides": [
    { "index": 1, "type": "cover", "title": "2026 사업 보고서" },
    { "index": 2, "type": "toc", "title": "목차" },
    { "index": 3, "type": "section-divider", "title": "01. 경영 실적 요약" },
    { "index": 4, "type": "key-metrics", "title": "핵심 지표 요약" },
    { "index": 5, "type": "chart-bar", "title": "연도별 매출 추이" },
    { "index": 6, "type": "data-table", "title": "사업부별 실적 비교" }
  ]
}
```

**유형 배정 규칙:**
- 숫자/지표/KPI 언급 → `key-metrics` 또는 `chart-bar`/`chart-pie`
- 비교/대비 언급 → `two-column`
- 절차/과정 언급 → `process-flow`
- 시간순/로드맵 → `timeline`
- 상세 설명 3개 이상 → `bullet-detail`
- 표/데이터 구조 → `data-table`
- 섹션 시작 → `section-divider`

### Step 2: 슬라이드 콘텐츠 생성

각 슬라이드의 **실제 콘텐츠**를 JSON으로 생성합니다.

```json
{
  "index": 4,
  "type": "key-metrics",
  "title": "핵심 지표 요약",
  "subtitle": "2026 Q1 기준",
  "background": "dark",
  "content": {
```

## PPTX 생성 규칙

### 슬라이드 마스터 설정

```javascript
// 16:9 비율
pptx.defineLayout({ name: "DECK", cx: 13.333, cy: 7.5 });

// 마스터 슬라이드 2종
pptx.defineSlideMaster("DARK_BG", {
  background: { color: "1B1F3B" },
  objects: [
    // 하단 페이지 번호
    { text: { text: "{{slideNumber}}", options: { x: 12.5, y: 7.0, fontSize: 9, color: "94A3B8" } } }
  ]
});

pptx.defineSlideMaster("LIGHT_BG", {
  background: { color: "F1F5F9" },
  objects: [
    { text: { text: "{{slideNumber}}", options: { x: 12.5, y: 7.0, fontSize: 9, color: "64748B" } } }
  ]
});
```

### 차트 생성 규칙

- 막대 차트: `pptx.addChart(pptx.charts.BAR, data, opts)`
- 파이/도넛: `pptx.addChart(pptx.charts.DOUGHNUT, data, opts)`
- 차트 색상: ACCENT → ACCENT_ALT → SUCCESS → WARNING 순환
- 폰트: Pretendard (없으면 fallback: "맑은 고딕", "Malgun Gothic")
- 차트 크기: 슬라이드 영역의 60~70% 차지

### 테이블 생성 규칙

- 헤더행: SECONDARY 배경, WHITE 텍스트, Bold
- 데이터행: 짝수행 SURFACE, 홀수행 WHITE
- 강조셀 (증감률 등): 양수 → SUCCESS, 음수 → DANGER
- 테두리: BORDER 색상, 0.5pt
- 여백: 셀 padding 0.05인치

### 텍스트 규칙

- **줄 간격**: 1.2~1.4 (밀집하되 가독성 유지)
- **단락 간격**: before 0.08인치, after 0.05인치
- **불릿**: "▪" 사용, ACCENT 색상
- **강조**: Bold + ACCENT 색상 (밑줄 사용 금지)
- **숫자 강조**: 36~48pt Bold (주변 텍스트 대비 2~3배 크기)

## 디자인 가이드라인

### 정보 밀도 규칙

1. **빈 공간 비율 ≤ 25%** — 화면의 75% 이상을 콘텐츠로 채움
2. **슬라이드당 100~200단어** — 키워드가 아니라 **문장**으로 전달
3. **데이터는 반드시 시각화** — 숫자만 나열하지 말고 차트/KPI 카드로
4. **계층 구조 명확화** — 제목/소제목/본문 3단계 폰트 크기 차이
5. **색상으로 구분** — 카테고리별 색상 코딩, 증감 표시(SUCCESS/DANGER)

### 레이아웃 규칙

6. **첫 슬라이드 = cover** 필수
7. **마지막 슬라이드 = closing** 필수
8. **5슬라이드 이상이면 toc 포함**
9. **10슬라이드 이상이면 section-divider로 섹션 구분**
10. **연속 3개 같은 유형 금지**
11. **슬라이드 수: 목차 항목 수의 1.5~2배** (목차 10개 → 15~20 슬라이드)
12. **통일 다크 배경** — 모든 슬라이드 동일 BG, 카드 SURFACE로 계층 구분

### 콘텐츠 규칙

13. **모든 차트에 출처/기준 시점 명시**
14. **숫자에는 단위 필수** (%, 억원, 명 등)
15. **비교 데이터는 기준 명시** (전년 대비, 전분기 대비 등)
16. **각 슬라이드 하단에 핵심 인사이트 1줄**
17. **약어 첫 등장 시 풀네임 병기**
