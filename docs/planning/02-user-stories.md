# 사용자 스토리 (User Stories)

## 개요

newVideoGen 프로젝트의 사용자 스토리를 기능 영역별로 정리했습니다. 각 스토리는 **"~로서, ~을 원한다, 왜냐하면 ~이기 때문이다"** 형식으로 작성되었으며, 우선순위(P0/P1/P2)를 부여했습니다.

---

## 🔴 P0 (필수, MVP에 포함)

### US-1: 자동 프레젠테이션 영상 생성

**As a** 유튜브 콘텐츠 크리에이터
**I want to** SRT 자막과 더빙 오디오를 폴더에 놓고 Claude Code 스킬을 실행하면, 레퍼런스 수준(80%+)의 프레젠테이션 영상이 자동으로 생성되기를
**Because** 현재 영상 편집에 수시간이 소요되므로, AI가 제안한 구성으로 빠르게 제작하고 싶다

**수락 기준**:
- [ ] SRT 파일 입력 가능 (자막 경로 지정)
- [ ] 오디오 파일 입력 가능 (mp3/wav)
- [ ] 최종 mp4 파일 생성됨
- [ ] 제작 시간 수시간 → 수십분 단축
- [ ] AI 결과물 정확도 80%+

**기술 요구사항**:
- 의미 청킹 엔진 (/vg-chunk)
- 장면 선택 엔진 (/vg-scene)
- Remotion 렌더링 (/vg-render)

---

### US-2: 웹 타임라인 에디터에서 결과 미리보기 및 편집

**As a** 유튜브 콘텐츠 크리에이터
**I want to** AI가 제안한 장면 구성을 웹 타임라인 에디터에서 시각적으로 확인하고, 필요하면 구간을 나누거나 병합하거나 템플릿을 변경할 수 있기를
**Because** AI 결과가 항상 완벽하지는 않으므로, 최종 확정 전에 미세 조정하고 싶다

**수락 기준**:
- [ ] 웹 브라우저에서 타임라인 에디터 열기 가능
- [ ] 각 장면 카드가 나열되고, 선택 시 상세 정보 표시
- [ ] 구간 나누기/병합 버튼 작동
- [ ] 템플릿 변경 드롭다운 작동
- [ ] Remotion Player로 실시간 미리보기 가능

**기술 요구사항**:
- Next.js 웹 앱 (S1 화면)
- Timeline/SceneCard/TemplateSelector 컴포넌트
- RemotionPlayer 통합

---

### US-3: 자막 타이밍과 애니메이션 동기화

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 자막이 화면에 나타나는 순간에 정확히 맞춰 텍스트, 배경, 로고 등의 요소가 애니메이션되기를
**Because** 정확한 타이밍이 시청자의 이해도와 시청 경험을 크게 좌우한다

**수락 기준**:
- [ ] 오디오 파형과 자막이 동기화되어 표시됨
- [ ] 비트 마커가 자막 타이밍 위치에 표시됨
- [ ] 각 장면의 애니메이션이 비트 단위로 정렬됨
- [ ] 프리뷰에서 오디오와 애니메이션이 동시 재생됨

**기술 요구사항**:
- beats.json 스키마 (비트 정보 포함)
- Scene DSL의 타이밍 정보 (duration, delay, stagger)
- Remotion Player 동기화

---

### US-4: 반복되는 장면 최소화

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 같은 템플릿/레이아웃이 계속 반복되지 않기를
**Because** 반복되는 시각 요소는 시청자 피로감을 유발하고 전문성을 떨어뜨린다

**수락 기준**:
- [ ] 8개 루트 레이아웃 패밀리가 균등하게 분포 (반복률 < 10%)
- [ ] 연속된 같은 패밀리는 최대 1회만 허용
- [ ] 최근 사용한 레이아웃을 다시 사용할 때 패널티(-25점) 적용
- [ ] 직전 장면과 유사도가 높으면 패널티(-20점) 적용

**기술 요구사항**:
- 장면 선택 엔진의 제약 기반 변주
- 반복 패널티 시스템
- 유사도 계산 알고리즘

---

## 🟡 P1 (중요, Phase 1-2 완료 후)

### US-5: 의미 역할별 자동 장면 선택

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 자막의 의미 역할(주장/비교/사례/경고/과정/요약)을 자동으로 감지하고, 그에 맞는 시각적 구조(레이아웃)가 자동으로 선택되기를
**Because** 수동으로 각 구간마다 적절한 템플릿을 선택하는 작업을 건너뛰고 싶다

**수락 기준**:
- [ ] 의미 청킹 엔진이 각 구간의 intent 태그 추출
- [ ] intent 별 권장 레이아웃 매핑 (예: 비교 → split-2col)
- [ ] 의미적합도 점수 계산 (40점)
- [ ] 정확도 80%+ 달성

**기술 요구사항**:
- /vg-chunk 스킬 (intent 추출)
- /vg-scene 스킬 (레이아웃 매핑)
- 의미-템플릿 규칙 카탈로그

---

### US-6: 증거 타입 기반 시각화 선택

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 자막이 언급하는 증거 타입(통계/인용/시각화/프로세스)에 따라 적절한 그래픽(바 차트/라인 차트/플로우 다이어그램)이 자동 선택되기를
**Because** 각 주장을 뒷받침하는 가장 효과적인 시각 표현이 자동으로 적용되어야 한다

**수락 기준**:
- [ ] evidenceType (통계/인용/시각화/프로세스) 추출
- [ ] evidenceType 별 권장 그래픽 (바 차트, 라인 차트, 다이어그램 등)
- [ ] 그래픽 자산이 실제로 렌더링됨
- [ ] 적합도 점수 (20점) 계산

**기술 요구사항**:
- beats.json의 evidenceType 필드
- 그래픽 컴포넌트 라이브러리 (BarCompare, LineChart, ProcessFlow 등)
- 증거-그래픽 매핑 규칙

---

### US-7: 정보 밀도와 비트 관리

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 정보 밀도가 높은 구간은 더 많은 애니메이션 비트(여러 단계)로 표현되고, 낮은 구간은 단순하게 표현되기를
**Because** 복잡한 정보는 단계적으로 전달해야 이해하기 쉽다

**수락 기준**:
- [ ] density (low/medium/high) 추출
- [ ] beatCount 자동 계산 (밀도 기반)
- [ ] 리듬적합도 점수 (15점) 계산
- [ ] 프리뷰에서 단계적 애니메이션 확인

**기술 요구사항**:
- beats.json의 density, beatCount 필드
- Scene DSL의 계층적 애니메이션 (staggerChildren)
- 정보-비트 매핑 규칙

---

### US-8: 로고/브랜드 자산 자동 통합

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 특정 회사나 인물, 제품명이 언급될 때 자동으로 해당 로고/아이콘이 화면에 표시되기를
**Because** 시각적 일관성과 브랜드 인식을 강화하고 싶다

**수락 기준**:
- [ ] 로고/아이콘 메타데이터 등록 (이름 ↔ SVG 파일)
- [ ] 자막에서 엔티티(회사명/인물명/제품명) 추출
- [ ] 엔티티와 로고/아이콘 매핑
- [ ] Scene DSL에 자산 호출 명령 자동 포함

**기술 요구사항**:
- NER(Named Entity Recognition) 또는 키워드 매칭
- assets/brands, assets/icons 카탈로그
- Asset Grammar (Asset DSL)

---

### US-9: 실시간 프리뷰와 프리스크럽

**As a** 유튜브 콘텐츠 크리에이터
**I want to** Remotion Player에서 장면 단위로 네비게이션하고, 특정 프레임으로 프리스크럽할 수 있기를
**Because** 빠르게 여러 지점을 확인하고 수정 여부를 판단하고 싶다

**수락 기준**:
- [ ] Remotion Player 통합 (S2 화면)
- [ ] Play/Pause/Stop 컨트롤
- [ ] 장면 이전/다음 버튼
- [ ] 타임라인 드래그로 프리스크럽
- [ ] 재생 속도 조절 (0.5x, 1x, 1.5x, 2x)

**기술 요구사항**:
- Remotion Player API
- RemotionPlayer/SceneNavigation/PlaybackControls 컴포넌트
- Scene DSL과 Remotion 렌더링 연동

---

## 🟢 P2 (선택, Phase 3 이후 고려)

### US-10: 커스텀 컬러 팔레트 지원

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 브랜드 색상 규칙에 맞춰 다크/네온 그린 외 다른 팔레트를 지정할 수 있기를
**Because** 시리즈별로 색상을 다르게 적용하고 싶다

**수락 기준**:
- [ ] 프로젝트 설정에서 기본 색상, 강조색 지정 가능
- [ ] design-tokens.json에 커스텀 팔레트 저장
- [ ] Scene DSL 렌더링 시 토큰 참조

**기술 요구사항**:
- design-tokens.json 확장
- Tailwind 색상 변수 동적 로드
- 프로젝트 설정 UI

---

### US-11: 다국어 자막 지원

**As a** 국제 콘텐츠 크리에이터
**I want to** 같은 영상에 대해 한국어, 영어, 일본어 등 여러 언어의 SRT를 생성할 수 있기를
**Because** 글로벌 시청자를 확보하고 싶다

**수락 기준**:
- [ ] 언어별 SRT 파일 입력 지원
- [ ] 각 언어별 의미 청킹 (다국어 NLP 필요)
- [ ] 텍스트 길이 정규화 (언어별 문자 길이 차이)
- [ ] 다국어별 렌더링

**기술 요구사항**:
- 다국어 NLP 모델
- 번역 API 통합 (선택)
- 다국어 폰트 지원

---

### US-12: 톤/스타일 커스터마이제이션

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 같은 의미의 자막이더라도 '강조' vs '설명' 톤에 따라 다른 애니메이션 스타일이 적용되기를
**Because** 콘텐츠의 맥락(강조 필요 vs 단순 설명)에 맞춰 표현하고 싶다

**수락 기준**:
- [ ] tone 필드 추출 (강조/설명/증명/반전)
- [ ] tone별 모션 프리셋 (강조 → popNumber, 설명 → fadeUp)
- [ ] Scene DSL에 tone 정보 포함
- [ ] 프리뷰에서 톤별 애니메이션 확인

**기술 요구사항**:
- beats.json의 tone 필드
- 톤-애니메이션 매핑
- Motion Grammar 확장

---

### US-13: 템플릿 라이브러리 관리

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 자주 쓰는 Scene DSL 조합을 '템플릿'으로 저장하고, 다른 영상에서 재사용할 수 있기를
**Because** 일관된 스타일 유지와 반복 작업 시간을 절약하고 싶다

**수락 기준**:
- [ ] 현재 장면을 '템플릿으로 저장' 버튼
- [ ] 저장된 템플릿 목록 표시
- [ ] 템플릿 적용 (Scene DSL 복사 또는 상속)
- [ ] 템플릿 삭제/수정/분류

**기술 요구사항**:
- 사용자 템플릿 메타데이터 (이름, 설명, 태그)
- 파일 시스템 또는 DB 저장
- 템플릿 관리 UI

---

### US-14: 분석 및 리포팅

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 생성된 영상의 통계(장면 분포, 애니메이션 비율, 렌더링 시간 등)를 대시보드에서 확인할 수 있기를
**Because** 영상의 구성 패턴을 이해하고, 향후 개선점을 찾고 싶다

**수락 기준**:
- [ ] 대시보드 화면 (S4 추가)
- [ ] 레이아웃 분포 차트 (8개 패밀리별 사용 비율)
- [ ] 애니메이션 시간 분석
- [ ] 렌더링 성능 그래프

**기술 요구사항**:
- 분석 데이터 수집 (scene-plan.json 기반)
- 차트 라이브러리 (recharts, D3 등)
- 대시보드 UI

---

### US-15: 배치 처리 (여러 영상 동시 생성)

**As a** 유튜브 콘텐츠 크리에이터
**I want to** 여러 SRT + 오디오 쌍을 한 번에 제출하면, 모두 병렬로 처리되기를
**Because** 시리즈 콘텐츠를 빠르게 생성하고 싶다

**수락 기준**:
- [ ] 배치 입력 폴더 지정
- [ ] 병렬 처리 (의미청킹, 장면선택, 렌더링)
- [ ] 진행률 모니터링
- [ ] 실패 항목 재시도

**기술 요구사항**:
- 큐 시스템 (BullMQ 등)
- 병렬 처리 (Worker)
- 상태 추적 (pending, processing, completed, failed)
