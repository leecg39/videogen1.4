# PRD (Product Requirements Document)
newVideoGen - AI 기반 자동 프레젠테이션 영상 생성 도구

## 📋 MVP 캡슐

| 항목 | 내용 |
|------|------|
| **목표** | SRT 자막 + 더빙 오디오를 입력으로 받아, AI가 자동으로 프레젠테이션 스타일 영상 생성 |
| **페르소나** | 유튜브 콘텐츠 크리에이터 (본인) |
| **핵심 기능** | ① 의미 청킹 ② 장면 선택 + DSL 생성 ③ 타임라인 에디터 ④ Remotion 렌더링 |
| **North Star** | 영상 제작 시간: 수시간 → 수십분 |
| **성공 지표** | AI 장면 매핑 정확도 80%+, 장면 반복률 최소화 |
| **핵심 리스크** | 의미 청킹 → 템플릿 매핑 정확도 |
| **완화 전략** | 레퍼런스 기반 규칙 정교화 + 제약 기반 변주 시스템 |

## 🎯 프로젝트 정의

### 문제 정의

유튜브 콘텐츠 크리에이터는 음성 스크립트를 더빙한 후, SRT 자막을 기반으로 시각적으로 매력적인 프레젠테이션 영상을 만들어야 한다. 현재 프로세스:
1. SRT 자막 생성 (수동 또는 Vrew)
2. 레퍼런스 영상 분석 (어떤 장면을 쓸지)
3. 편집 소프트웨어에서 수시간 동안 각 구간마다 장면, 텍스트, 모션 설정
4. 렌더링

이는 **수시간의 반복적 수작업**을 필요로 한다.

### 솔루션 제안

**newVideoGen**: Claude Code 스킬 + Next.js 웹 에디터 + Remotion 렌더링으로 다음을 달성:
- **의미 청킹**: SRT → 의미 역할(intent) + 비트(beat) 추출
- **장면 선택**: 의미적 적합도 기반 제약 변주로 8개 레이아웃 패밀리 중 선택
- **웹 에디터**: AI 제안을 시각적으로 검토 + 미세 조정
- **1회성 렌더링**: Scene DSL → Remotion TSX → mp4

- **통증점**:
  - SRT → 영상 변환 수시간 소요
  - 반복되는 템플릿 선택 작업
  - 자막 타이밍과 애니메이션 동기화 어려움
- **목표**: 품질 저하 없이 제작 시간을 수십분으로 단축

## 🎬 핵심 기능

### FEAT-1: 의미 청킹 엔진

**입력**: SRT 파일
**출력**: beats.json

각 구간을 의미 단위로 청킹하고:
- **intent** (주장/비교/사례/경고/과정/요약)
- **tone** (강조/설명/증명/반전)
- **evidenceType** (통계/인용/시각화/프로세스)
- **emphasisTokens** (강조할 키워드 배열)
- **density** (정보 밀도: low/medium/high)
- **beatCount** (비트 수: 몇 초에 걸친 장면인가)

### FEAT-2: 장면 선택 + DSL 생성 엔진

**입력**: beats.json + 8개 루트 레이아웃 패밀리 카탈로그
**출력**: scene-plan.json + scene-[id].json (Scene DSL)

점수 시스템:
```
최종 점수 = 의미적합도(40)
          + 증거타입적합도(20)
          + 리듬적합도(15)
          + 자산보유(10)
          - 최근중복패널티(25)
          - 직전유사도(20)
```

제약 기반 변주로 동일 레이아웃의 반복을 최소화.

Scene DSL = JSON 형식의 중간 표현
- 레이아웃 타입 (hero-center, split-2col 등)
- 컴포넌트 트리 (kicker, headline, supporting text 등)
- 모션 시퀀스 (fadeUp, popNumber 등)
- 타이밍 (비트 단위 애니메이션)

### FEAT-3: 타임라인 에디터 (웹 앱)

**입력**: scene-plan.json + 오디오 파일
**출력**: 편집된 scene-*.json + mp4 (최종)

웹 UI:
- 상단: 프로젝트 선택 / SRT 파일 경로 / 기본 설정
- 중앙: 수평 스크롤 타임라인 (장면 카드 배열)
- 우측 패널: 선택된 장면의 DSL JSON 편집 / 템플릿 변경
- 하단: 오디오 파형 + 비트 마커 (동기화 확인)

기능:
- 구간 나누기/병합
- 템플릿 변경
- DSL 수정
- Remotion Player 실시간 프리뷰

### FEAT-4: Remotion 렌더링

**입력**: 편집된 scene-*.json
**출력**: mp4 파일

Scene DSL → Remotion TSX 컴포넌트 자동 생성
- 스택 레이아웃 엔진: 상대 좌표 계산
- 모션 프리셋: fadeUp, staggerChildren, popNumber, countUp 등
- 최종 렌더링: Remotion 렌더러로 mp4 생성

## 📊 성공 지표 (OKR)

### Objective: "AI 기반 자동 프레젠테이션 영상 생성으로 제작 시간 단축"

| KR | 목표값 | 측정 방법 |
|----|----|------|
| **KR1** | AI 장면 매핑 정확도 80%+ | 레퍼런스 영상과 비교, 템플릿 오류율 |
| **KR2** | 동일 장면 반복률 < 10% | scene-*.json 분석 (8개 패밀리 분포) |
| **KR3** | 제작 시간 수시간 → 수십분 | 매 영상마다 측정 (의미청킹 + 렌더링 시간) |
| **KR4** | 사용자 만족도 4/5 이상 | 웹 에디터 UX + 프리뷰 품질 |

## 🎯 범위 (Scope)

### In Scope

- ✅ SRT 자막 → 의미 청킹
- ✅ 의미 → 장면 DSL 자동 선택
- ✅ 웹 기반 타임라인 에디터
- ✅ Remotion 렌더링 (mp4 출력)
- ✅ Claude Code 스킬 시스템 (5개 스킬)
- ✅ 8개 루트 레이아웃 패밀리 (hero-center, split-2col, grid-4x3, process-horizontal, radial-focus, stacked-vertical, comparison-bars, spotlight-case)

### Out of Scope

- ❌ SRT 파일 자동 생성 (Vrew 활용)
- ❌ 음성 스크립트 작성
- ❌ 더빙 오디오 생성
- ❌ 자산(로고/아이콘) 설계
- ❌ 추가 AI 모델 학습

## 🚨 리스크 & 완화 전략

| 리스크 | 영향 | 확률 | 심각도 | 완화 전략 |
|--------|------|------|--------|----------|
| 의미 청킹 정확도 < 70% | AI 결과물 품질 저하 | 높음 | 높음 | 레퍼런스 영상 기반 규칙 정교화, 테스트 케이스 다양화 |
| 장면 반복으로 영상 단조로움 | 콘텐츠 품질 저하 | 높음 | 중간 | 제약 기반 변주 + 반복 페널티(-25점) |
| Scene DSL 스키마 변경 필요 | 개발 지연 | 중간 | 중간 | 8개 레이아웃 먼저 구현, 점진적 확장, 버전 관리 |
| Remotion 렌더링 성능 저하 | 제작 시간 증가 | 중간 | 중간 | 프리뷰 최적화, 캐싱 전략, 단계적 렌더링 |
