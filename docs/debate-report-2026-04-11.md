**주제:** newVideoGen 프로젝트의 본질, 강점, 결함, 전략적 방향

---

## 1. 프로젝트 정의 (3자 합의)

> SRT/오디오를 입력으로 받아 AI가 stack_root JSON 트리(DSL)를 생성하고, Remotion(React)이 이를 물리적 영상으로 렌더링하는 **데이터 드리븐 영상 자동 생성 엔진**.

단순 템플릿이 아니라 동적 레이아웃 최적화를 지향하며, 파이프라인은 다음과 같다:

```
자막 분석(Beats) → 레이아웃 생성(AI) → 싱크 최적화 → Remotion 렌더링
```

## 2. 강점 (합의 영역)

| 영역 | 상세 | 평가자 |

| **단계별 최적화 파이프라인** | Beats → Scene Plan → Layout → Render로 이어지는 정교한 단계 분리 | Gemini |
| **렌더링 비용 우위** | Gen-AI 대비 압도적으로 저렴한 렌더링 비용, 수술적 수정 가능 | Gemini |

## 3. 기술 부채 (논쟁 + 합의)

### 3-1. 합의된 핵심 부채

| 우선순위 | 부채 | 상세 |
|----------|------|------|
| **P0** | 타입-스키마 불일치 | schemas.ts enum이 types/index.ts보다 뒤처져 검증 누락. NodeProps Generic 실패로 data/layout이 unknown 처리 → 컴파일 타임 검증 불가 |
| **P1** | 좀비 컴포넌트 잔존 | InsightTile이 CLAUDE.md에서 금지이나 registry.ts + 생성 스크립트에 잔존. 런타임에 빈 박스 렌더링 위험 |
| **P2** | 사후 보정 스크립트 의존 | fix-transition-drift.ts, sync-enterAt.ts 등 파편화된 보정 로직. AI 생성 JSON과 React 렌더러 간 정밀 규약 부재가 근본 원인 |
| **P3** | 타이밍 동기화 취약 | enterAt/자막/씬 전환이 프레임/초 단위로 혼재 |

### 3-2. 근본 원인 진단

**Codex:** "유연한 DSL/JSON 구조에 비해 타입·스키마·검증이 구현과 분리된 상태" → 모든 하위 부채의 연쇄적 원인

**Gemini:** "NodeProps의 범용화(Generic) 실패가 핵심" + "AI 생성 JSON과 React 렌더러 간 정밀 규약 부재"

**Claude:** 두 진단 모두 같은 문제의 다른 층위를 가리키고 있음. Codex는 시스템 수준, Gemini는 타입 수준에서 동일한 결론에 도달.

## 4. "AI가 만든 것 같지 않은 영상" — 목표 달성 가능성

### 3자 평가 요약

| 평가자 | 판정 | 핵심 논리 |
|--------|------|-----------|
| **Codex** | 부분 달성 가능 | 규칙 기반 조합의 냄새를 완전히 숨기기 어렵고, 인간급 미학 판단 루프 없이는 구조적 한계 |
| **Gemini** | Space 90%, Time 60% | 레이아웃 제어는 탁월하나 나레이션의 미묘한 뉘앙스에 반응하는 '시각적 호흡'은 정량화 극난. AI는 '고급스러운 기성품'은 가능해도 '의도적 파격(Contextual Subversion)' 재현에 구조적 한계 |
| **Claude** | Variant 시스템 + 프레임 카탈로그가 시스템화 시도이나 불충분 | '규칙적 다양성'은 여전히 규칙 안의 변주. "판단"이 빠져 있음 |

### 합의 결론

> **현재 시스템은 '반복 회피'는 달성하지만 '의도적 파격'은 미달.  
> 조합 수(variant)보다 "판단 시스템"이 부족하다.**

### 포지셔닝 합의

**최종 합의:** 두 전략은 양립 가능. "완성 영상 생성기"보다 **"구조·타이밍·브랜드를 통제하는 오케스트레이터"**로의 진화가 생존 전략.

## 6. 추천 액션 (우선순위)

### 즉시 실행 (P0)

1. **Zod 스키마 단일화** — schemas.ts ↔ stack-nodes.ts 동기화, 노드별 데이터 스키마 강제
2. **좀비 코드 제거** — InsightTile 등 금지 노드를 레지스트리 + 생성 스크립트에서 완전 제거

### 단기 (P1)

3. **자동 심사 레이어** — 렌더 전 scene score를 계산하여 미학 규칙 위반 시 variant 재선택/재생성
4. **타이밍 단위 통일** — 프레임 기준으로 enterAt/전환 단위 통일

### 중기 (P2)

5. **세만틱 리듬 엔진** — 나레이션 강조어 + 오디오 파형 분석 → 노드 등장 속도/트랜지션 템포 실시간 조정
6. **보정 스크립트 내재화** — fix-*.ts를 stack-composer 또는 렌더링 전처리 파이프라인으로 통합

### 장기 (P3)

7. **Gen-AI B-roll 오케스트레이션** — 배경 에셋으로 Gen-AI 영상 자동 삽입·평가·교체
8. **카오스 팩터** — 텍스트 정서적 강도에 따른 레이아웃 규칙 의도적 위반

## 9. 핵심 결론 (3자 합의)

> **이 프로젝트의 승부처는 "생성"이 아니라 "판단과 오케스트레이션"이다.**
>
> DSL 기반 정밀도는 이미 확보됨. 부족한 것은 두 가지:
> 1. **내실** — 타입 안전성, 스키마 동기화, 좀비 코드 제거
> 2. **감성** — 세만틱 리듬 엔진으로 '인간적 호흡' 구현
>
> 이 두 축이 완성되면, Gen-AI 시대에서 **정보 영상의 오케스트레이터**로서 독보적 위치를 확보할 수 있다.
