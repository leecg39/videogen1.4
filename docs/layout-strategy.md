# Layout Strategy — REF 라이브러리 활용 전략

## 핵심 원칙

> REF-001~070은 **"이 JSON을 복사하라"가 아니라 "이런 시각 언어가 존재한다"**를 알려주는 레퍼런스.
> 원본 영상의 내용(Claude Code, 토큰 등)이 아닌 **구조적 패턴과 시각 기법**만 추출하여 재조합한다.

---

## 1. REF를 직접 복사하지 않는 3가지 장치

### 1A. 패턴 태그 시스템 (구조 분해)

REF를 통째로 참조하지 않고, **패턴 태그**로 분해하여 조합한다.

```
REF-067 = [icon-hero] + [equation-headline] + [comparison-table] + [richtext-summary]
REF-068 = [tier-cards] + [progressbar-meter] + [cta-pill]
REF-069 = [before-after-cards] + [illustration-mockup] + [slide-motion]
REF-070 = [circular-image] + [keyword-hero] + [step-badge]
```

**활용**: 씬 분석 후 필요한 태그를 골라 새로운 조합을 만든다.
- "통계 비교 + 프로세스" → `[tier-cards]` from REF-068 + `[arrow-flow]` from REF-005
- "인물 소개 + 인용" → `[circular-image]` from REF-070 + `[quote-block]` from REF-008

### 1B. 스타일 변주 레이어

같은 구조라도 **색상/크기/방향/모션을 변주**하여 다르게 보이게 한다.

| 변주 축 | 원본 | 변주 예시 |
|---------|------|----------|
| 색상 반전 | gray 좌 + green 우 | green 좌(현재) + amber 우(목표) |
| 방향 전환 | 좌→우 수평 | 상→하 수직, 또는 우→좌 역방향 |
| 비율 변경 | Split 1:1 | Split 1:2 또는 2:1 |
| 밀도 조절 | 3열 카드 | 2열 또는 4열로 증감 |
| 모션 교체 | scale-in | fade-in, slide-up, stagger |
| 크기 스케일 | Icon 80px | Icon 120px (더 임팩트) 또는 40px (보조) |

### 1C. 컨텍스트 재해석

REF의 **시각 구조**는 가져오되, **내용은 현재 씬의 narration/metadata에서 새로 추출**한다.

```
REF-067 원본: "식비 = 토큰 비용" (요리 비유)
재해석 예시: "시간 = 기회비용" (경제 비유) — 같은 등식 구조, 다른 내용
재해석 예시: "코드 = 설계도" (건축 비유) — 아이콘도 냄비→건물로 변경
```

## 2. 기존 시스템과의 통합 방안

### 현재 레이아웃 생성 파이프라인

```
씬 데이터 (scenes-v2.json)
    ↓
/vg-layout 스킬 (Claude가 직접 생성)
    ↓ 참조
아키타입 A~Z (26개 추상 패턴)     ← 기존
    +
REF-001~070 (70개 구체 레퍼런스)  ← 신규
    ↓
stack_root JSON 트리 생성
    ↓
후처리 스크립트 4종
    ↓
최종 scenes-v2.json
```

### 2A. 2단계 매칭 시스템

**1단계: 아키타입 선택** (기존 A~Z 유지)

씬의 `chunk_metadata`로 대분류를 먼저 결정:

| intent | 1차 아키타입 후보 |
|--------|-----------------|
| compare | C(VS대비), O(CompareCard), REF-066~067계열 |
| list | D(3열Grid), I(스텝카드), J(체크리스트) |
| sequence | E(프로세스), P(타임라인), Y(TimelineStepper) |
| define | B(풀블리드), K(단일카드), REF-070(히어로) |
| emphasize | A(히어로), G(Warning), H(인용) |
| example | F(FrameBox), L(Split), V(디바이스) |

**2단계: REF 스타일 적용** (신규)

1차 아키타입이 정해진 후, REF에서 **세부 시각 기법**을 참조:

```
아키타입 C(VS대비) 선택됨
  → REF-066 스타일: Pill 태그 클러스터 + 색상 차등 border
  → REF-067 스타일: 테이블 행 + 중앙 카테고리 Badge
  → REF-003 스타일: 수직 구분선 + IconCard
  → 최종 결정: 씬 내용에 가장 적합한 세부 기법 채택
```

### 2B. REF 패턴 인덱스 (의미 기반 검색)

| 패턴 카테고리 | 해당 REF | 핵심 기법 |
|-------------|---------|----------|
| **이항 대비** | 003, 048, 060, 066, 067, 069 | Split + 색상 코딩 (gray vs green) |
| **3열 나열** | 004, 047, 052, 068 | Grid/Stack(row) + 카드 반복 |
| **프로세스 흐름** | 005, 051, 054, 058, 064, 065 | ArrowConnector + 순차 노드 |
| **수치/통계** | 055, 057, 063, 068 | StatNumber + ProgressBar + RingChart |
| **히어로 임팩트** | 045, 050, 053, 070 | 큰 아이콘/이미지 + 키워드 Headline |
| **카드 내 목업** | 059, 060, 061, 069 | FrameBox 안에 노드 조합으로 UI 모사 |
| **리스트/체크** | 046, 052, 062 | BulletList + ProcessStepCard + Badge |
| **인용/CTA** | 008, 068, 069 | QuoteText, Pill(CTA), InsightTile |
| **테이블/행 반복** | 067 | Stack(row) 반복 + Divider 행구분 |
| **스택 칩/태그** | 047, 061, 066 | Pill 클러스터, stagger pop-in |
| **이미지 활용** | 049, 070 | ImageAsset (원형/사각) + border |
| **모션 특수** | 057(spin-ring), 061(stagger), 069(slide-right) | 비표준 모션 프리셋 |

### 2C. 다양성 강화 규칙 (기존 규칙 확장)

기존 `/vg-layout` 스킬의 다양성 규칙에 REF 활용 규칙을 추가:

```
기존 규칙:
  - 연속 3씬 같은 아키타입 금지
  - 전체 최소 5종 아키타입
  - 매 씬 최소 1개 비텍스트 요소

추가 규칙 (REF 기반):
  - 20씬 기준 최소 8개 다른 REF 카테고리 참조
  - 같은 REF 번호를 2회 이상 직접 참조 금지
  - REF에서 가져온 기법은 최소 1가지 변주 적용 (색상/방향/비율/모션)
  - 이항 대비 씬: REF-066(카드) vs REF-067(테이블) 교대 사용
  - 프로세스 씬: 수평(REF-005) vs 수직(REF-051) vs 팬아웃(REF-064) 순환
  - 히어로 씬: 아이콘(REF-053) vs 이미지(REF-070) vs SVG(REF-050) 교대
```

## 3. 실전 적용 예시

### 예시: "머신러닝 기초" 10씬 영상

| 씬 | 내용 | 1차 아키타입 | REF 기법 차용 | 변주 |
|----|------|------------|-------------|------|
| 0 | ML이란? | A(히어로) | REF-070 원형이미지 | 아이콘→뇌 SVG, scale-in |
| 1 | 지도학습 vs 비지도 | C(VS대비) | REF-066 카드+Pill | green=지도, blue=비지도 |
| 2 | 데이터→모델→예측 | E(프로세스) | REF-005 수평화살표 | 3단계, 아이콘 변경 |
| 3 | 정확도 95% | R(번호) | REF-057 회전링+StatNumber | amber 강조색 |
| 4 | 과적합 경고 | G(Warning) | REF-069 비포/애프터 | 좌=과적합 그래프, 우=정상 |
| 5 | 3가지 평가지표 | D(3열Grid) | REF-068 티어카드+ProgressBar | 3개 메트릭 바 |
| 6 | 훈련 파이프라인 | P(수직) | REF-051 아키텍처 허브 | 세로 타임라인+LineConnector |
| 7 | 실제 코드 예시 | W(터미널) | REF-063 TerminalBlock | Python 코드 스니펫 |
| 8 | 체크리스트 정리 | J(체크) | REF-062 번호+Badge쌍 | before/after 뱃지 추가 |
| 9 | 핵심 요약 | B(풀블리드) | REF-053 등식HL+InsightTile | "데이터 = 연료" |

**결과**: 10씬에 8개 아키타입 + 10개 다른 REF 기법 → 반복 없음.

## 4. /vg-layout 스킬 업데이트 방향

### 스킬 프롬프트에 추가할 섹션

```markdown
### REF 라이브러리 활용 (docs/layout-reference-library.md)

70개 레이아웃 레퍼런스가 등록되어 있다. 활용 원칙:

1. **참조하되 복사하지 않는다** — REF의 구조적 패턴만 추출
2. **패턴 태그로 분해** — 하나의 REF에서 일부 기법만 차용 가능
3. **반드시 1개 이상 변주** — 색상/방향/비율/모션 중 최소 1가지 변경
4. **같은 REF 2회 참조 금지** — 20씬 기준
5. **의미 기반 매칭** — intent/evidence_type → REF 카테고리 인덱스 참조

REF 카테고리 빠른 참조:
- 이항 대비: 003, 048, 060, 066, 067, 069
- 3열 나열: 004, 047, 052, 068
- 프로세스: 005, 051, 054, 058, 064, 065
- 수치/통계: 055, 057, 063, 068
- 히어로: 045, 050, 053, 070
- 목업/일러스트: 059, 060, 061, 069
- 리스트: 046, 052, 062
- 테이블: 067
- 이미지: 049, 070
```

## 5. 요약: 3줄 전략

1. **REF = 시각 어휘** — "이 레이아웃을 써라"가 아니라 "이런 기법이 가능하다"
2. **2단계 매칭** — 아키타입(A~Z)으로 대분류 → REF로 세부 기법 선택 → 변주 적용
3. **조합 > 복사** — REF-066의 Pill 클러스터 + REF-068의 ProgressBar = 새로운 레이아웃
